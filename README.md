# Casa Tile

A bilingual (RO + RU) static product catalog for ceramic and floor tiles. No build dependencies, no Node.js — just PowerShell and a browser.

## What's here

- **30 sample products** across 8 collections (marble, wood, concrete, terrazzo, subway, hexagon, stone, decorative)
- **Auto-generated SVG product images** (parametric, deterministic per product)
- **Filterable catalog** — collection, color, size, room, finish, material, price range
- **Sortable** — newest, price asc/desc, name
- **Fully responsive** — mobile (375px) → desktop
- **SEO-optimized** — server-rendered HTML, per-page meta tags, OpenGraph, Twitter cards, JSON-LD structured data (`Store` on home, `Product` on product pages, `ItemList` on catalog), `sitemap.xml` with hreflang alternates, `robots.txt`
- **Bilingual** — Romanian (default) at `/`, Russian at `/ru/`

## Tech stack

- Plain HTML5 / CSS3 / vanilla JS — zero JS framework, zero CSS framework
- PowerShell 5.1 build script (uses only `System.IO`, `System.Random`, `System.Web`)
- Google Fonts via `<link>` (Cormorant Garamond display)

Total page weight: ~50 KB HTML + 14 KB CSS + 3 KB JS + ~2 KB SVG per product image.

## Build

The build is a single PowerShell script — it reads the JSON data and writes every HTML page, every SVG, the sitemap, and `robots.txt`.

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

Pass `-SiteUrl` to set the canonical base URL (used in `<link rel="canonical">`, OG tags, sitemap):

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1 -SiteUrl 'https://yourdomain.com'
```

## Preview locally

```powershell
powershell -ExecutionPolicy Bypass -File .\serve.ps1 -Port 8080
# open http://localhost:8080/
```

## Editing content

- **Products:** edit `data/products.json` and rebuild. Each product has `slug` (unique URL key), `name.ro` / `name.ru`, `collection`, `pattern` (one of `marble`, `wood`, `concrete`, `terrazzo`, `subway`, `hexagon`, `stone`, `decorative`), `baseColor` / `accentColor` (hex, used to generate the SVG), and stock/spec fields.
- **UI strings:** edit `data/i18n.json` and rebuild.
- **Real photos instead of SVGs:** drop a JPG/PNG into `assets/images/products/<slug>.jpg`, change the image path in `build.ps1` (`Render-ProductCard` and `Build-ProductPage`), and rebuild.

## Deploy

Any static host works — no server runtime needed.

- **GitHub Pages, Netlify, Cloudflare Pages, Vercel:** point at the repo root, build command `powershell -ExecutionPolicy Bypass -File build.ps1` (or run locally and commit the generated files), publish directory `.`.
- **Plain web server (nginx, Apache):** copy the entire folder. Set `index.html` as the directory index. No URL rewrites needed.

## What's not here (out of scope of this build)

- Cart / checkout flow (the catalog is for browsing + inquiries; the contact page accepts product inquiries via `?product=<slug>` query param).
- Backend for the contact form — `assets/js/contact.js` simulates success. To make it real, replace the simulated handler with a `fetch()` to a Cloudflare Worker, Formspree, your own API, etc.
- Image optimization beyond SVG (which is already the right format for these tile patterns; if you swap to photo JPGs, run them through a compressor).
